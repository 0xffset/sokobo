This project doesn\textquotesingle{}t require any special command-\/line flags to build to keep things simple.

Here are the steps for building in release mode with a single-\/configuration generator, like the Unix Makefiles one:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/S\ .\ -\/B\ build\ -\/D\ CMAKE\_BUILD\_TYPE=Release}
\DoxyCodeLine{cmake\ -\/-\/build\ build}

\end{DoxyCode}


Here are the steps for building in release mode with a multi-\/configuration generator, like the Visual Studio ones:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/S\ .\ -\/B\ build}
\DoxyCodeLine{cmake\ -\/-\/build\ build\ -\/-\/config\ Release}

\end{DoxyCode}

This project doesn\textquotesingle{}t require any special command-\/line flags to install to keep things simple. As a prerequisite, the project has to be built with the above commands already.

The below commands require at least CMake 3.\+15 to run, because that is the version in which \href{https://cmake.org/cmake/help/latest/manual/cmake.1.html\#install-a-project}{\texttt{Install a Project}} was added.

Here is the command for installing the release mode artifacts with a single-\/configuration generator, like the Unix Makefiles one:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/-\/install\ build}

\end{DoxyCode}


Here is the command for installing the release mode artifacts with a multi-\/configuration generator, like the Visual Studio ones:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/-\/install\ build\ -\/-\/config\ Release}
